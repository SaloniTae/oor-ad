================================================================================
OOR-AD API — SECTION 2: CHANNEL MANAGEMENT
================================================================================
Base URL:   https://<your-host>/api/v1
Auth:       x-api-key header (see 01-keys.txt for auth + error model)
Scopes:     read -> channels:read | write -> channels:write (full-access '*' has both)
Doc source: server/src/api/routes/channels.js
================================================================================

A channel is a named live source. The returned `stream_reference` is PORTABLE:
a third party can point ANY player at `source_url`, or protect it with the
signed pin-based playback flow (see 03-playback.txt).

source_type values:  m3u8 | mp4 | mkv | youtube | direct_url
status values:       active | disabled

CHANNEL OBJECT:
    {
      "id": "53Bz4o3-3MdncTDx",
      "name": "Sports HD",
      "slug": "sports-hd-jpfh",
      "source_type": "m3u8",
      "source_url": "https://cdn.example.com/live.m3u8",
      "status": "active",
      "require_pin": true,
      "stream_reference": {
        "source_url": "https://cdn.example.com/live.m3u8",
        "source_type": "m3u8",
        "secured": true,
        "playback_generate": "/api/v1/playback/generate"
      },
      "created_at": 1783801000000
    }

================================================================================
POST /api/v1/channels
--------------------------------------------------------------------------------
Purpose:   Create a channel.
Scope:     channels:write
Body:
    {
      "name":        "Sports HD",          // required
      "source_type": "m3u8",               // required, one of the 5 types
      "source_url":  "https://.../live.m3u8", // required, valid URL
      "slug":        "sports-hd",           // optional (auto-generated if omitted)
      "require_pin": true                    // optional, gate playback behind a PIN
    }

Example:
    curl -X POST https://host/api/v1/channels \
      -H "x-api-key: adi_xxx" -H "Content-Type: application/json" \
      -d '{"name":"Sports HD","source_type":"m3u8","source_url":"https://cdn/live.m3u8","require_pin":true}'

Success (201): the CHANNEL OBJECT above (+ "request_id").

Errors:
    400 VALIDATION_ERROR  { ..., "field":"source_type" }   (bad/missing field)
    409 SLUG_TAKEN        { ..., "field":"slug" }

================================================================================
GET /api/v1/channels?limit=20&offset=0&status=active
--------------------------------------------------------------------------------
Purpose:   List channels (paginated, optional status filter).
Scope:     channels:read
Query:     limit (1-100, default 20), offset (default 0), status (active|disabled)

Example:
    curl "https://host/api/v1/channels?limit=20&status=active" -H "x-api-key: adi_xxx"

Success (200):
    {
      "items": [ { CHANNEL OBJECT }, ... ],
      "pagination": { "total": 42, "limit": 20, "offset": 0, "has_more": true },
      "request_id": "req_..."
    }

================================================================================
GET /api/v1/channels/{id}
--------------------------------------------------------------------------------
Purpose:   Get one channel.
Scope:     channels:read

Example:
    curl https://host/api/v1/channels/53Bz4o3-3MdncTDx -H "x-api-key: adi_xxx"

Success (200): CHANNEL OBJECT (+ request_id)
Error (404):   { "error_code":"CHANNEL_NOT_FOUND", "field":"id", "request_id":"..." }

================================================================================
PATCH /api/v1/channels/{id}
--------------------------------------------------------------------------------
Purpose:   Update name / source / pin requirement (any subset).
Scope:     channels:write
Body (at least one field):
    { "name": "Sports 4K", "source_type": "m3u8",
      "source_url": "https://.../new.m3u8", "require_pin": false }

Example:
    curl -X PATCH https://host/api/v1/channels/53Bz4o3-3MdncTDx \
      -H "x-api-key: adi_xxx" -H "Content-Type: application/json" \
      -d '{"name":"Sports 4K","source_url":"https://cdn/new.m3u8"}'

Success (200): updated CHANNEL OBJECT
Error (400):   VALIDATION_ERROR if body empty or a field is invalid

================================================================================
POST /api/v1/channels/{id}/disable
POST /api/v1/channels/{id}/enable
--------------------------------------------------------------------------------
Purpose:   Soft disable / re-enable (keeps the channel, flips status).
Scope:     channels:write

Example:
    curl -X POST https://host/api/v1/channels/53Bz4o3-3MdncTDx/disable -H "x-api-key: adi_xxx"

Success (200): CHANNEL OBJECT with "status":"disabled" (or "active" for enable)

================================================================================
GET /api/v1/channels/{id}/state   (live broadcast state + concurrent viewers)
--------------------------------------------------------------------------------
Purpose:   Render the channel's "● BROADCAST LIVE / ● COMMERCIAL BREAK" widget.
Scope:     channels:read
Combines the cluster-safe concurrent viewer count with the current ad-break
state (identical shape to GET /ads/state — live | bumper | ad + pod progress).

Example:
    curl https://host/api/v1/channels/53Bz4o3-3MdncTDx/state -H "x-api-key: adi_xxx"

Success (200) — LIVE:
    { "channel_id":"...", "concurrent_viewers":128, "state":"live",
      "is_ad_break":false, "request_id":"..." }
Success (200) — DURING A BREAK: same, plus the bumper/current_ad/pod fields
    documented in 04-ads.txt (GET /ads/state).
Error (404): CHANNEL_NOT_FOUND

================================================================================
GET /api/v1/channels/{id}/triggers?limit=50&offset=0   (ad-break history)
--------------------------------------------------------------------------------
Purpose:   The "Recent triggers" table (when / duration / status).
Scope:     channels:read
Query:     limit (1-200, default 50), offset (default 0)

Example:
    curl "https://host/api/v1/channels/53Bz4o3-3MdncTDx/triggers?limit=50" -H "x-api-key: adi_xxx"

Success (200):
    { "items":[ { "id":"...", "ad_id":"...", "status":"completed",
                  "duration_seconds":12, "start_at":..., "end_at":...,
                  "triggered_by":"api", "pod":[...], "is_error":false,
                  "created_at":... }, ... ],
      "pagination":{ "limit":50, "offset":0, "returned":3 }, "request_id":"..." }
    - is_error=true and status="error:*" when a viewer reported a playback failure.
Error (404): CHANNEL_NOT_FOUND

================================================================================
POST /api/v1/channels/{id}/viewer-token   (issue a viewer token + player links)
--------------------------------------------------------------------------------
Purpose:   "Get viewer URL" / "Open player" / "Copy embed code" — everything a
           third party needs to drop ONE viewer onto the channel.
Scope:     playback:write
Body (all optional):
    { "viewer_id":"seat-42", "ttl_seconds":3600 }
    - viewer_id  stable id for this seat (auto-generated if omitted)
    - ttl_seconds 60..86400, default 3600

Example:
    curl -X POST https://host/api/v1/channels/53Bz4o3-3MdncTDx/viewer-token \
      -H "x-api-key: adi_xxx" -H "Content-Type: application/json" -d '{}'

Success (200):
    { "channel_id":"...", "viewer_id":"c0qgyrwCc94",
      "token":"eyJ...", "expires_in":3600,
      "ws_url":"wss://host/ws?channel=<slug>&token=<jwt>",
      "player_url":"https://host/player/?ws=...(&secure=1&ch=<slug> if PIN-gated)",
      "embed_code":"<iframe src=... allowfullscreen></iframe>",
      "secured":false, "request_id":"..." }
    - secured=true when the channel requires a PIN; then player_url carries
      secure=1 so the PIN gate engages (mint the PIN via /playback/generate or
      /pins — see 03-playback.txt / 06-pins.txt).
Error (404): CHANNEL_NOT_FOUND

================================================================================
DELETE /api/v1/channels/{id}
--------------------------------------------------------------------------------
Purpose:   Permanently delete a channel.
Scope:     channels:write

Example:
    curl -X DELETE https://host/api/v1/channels/53Bz4o3-3MdncTDx -H "x-api-key: adi_xxx"

Success (200): { "id":"53Bz4o3-3MdncTDx", "deleted":true, "request_id":"req_..." }
Error (404):   CHANNEL_NOT_FOUND
